Inline

  • 줄바꿈 없음
  • a, img, span
  • width, height로 크기 조절 X(=레이아웃에 활용 불가)
  • 좌우 margin, 좌우 padding 만 적용 가능
  • display:inline;

Block

  • 줄바꿈 있음
  • h, div, p
  • width, height로 크기 조절 가능
  • 내용과 별개로 자신의 영역을 가짐(내용을 삭제 해도 영역은 남아 있음
  • display:block;

inline-block

  • 줄바꿈 없음
  • block 요소 처럼 width, height 가짐
  • 상하 margin, padding도 존재
  • display:inline-block;

Float

자식 요소가 부모 요소로 부터 떠오른다(margin 제거)
block 요소들은 width 지정 시 자식 요소의 width 값 자체가 변하는 게 아니라 부모 요소의 width 만큼 float가 적용된 width를 제외한 너비에 margin이 생겨서 그렇게 보이는 것임
float 요소와 비float 요소를 사용 시 비 float 요소가 아래 층에 깔림
만약 가로 배치 하고 싶은 요소가 있다면 전부 float 선언

float:left; right, center 도 존재

Clear

Float로 띄운 것때문에 margin이 사라져서 텍스트 끼리 겹쳐지는 현상(범람)을 방지하고자 생긴 속성
Float 된 애들 밑으로 안 가게 할 요소에 Clear:left를 적용
Margin이 다시 생긴 게 아니라 위의 margin 0에 영향을 받지 않겠다 선언 하는 느낌?인?듯? float의 속성 값에 영향을 받지 않으려면(=기능이 정상 작동 하려면) float와 동일 값(float:left인 경우 clear:left)를 적용 해야함

Clearfix : Clear 속성으로 float의 layout을 바로 잡는 기법
범람을막고싶은요소이름:: after { // after 어떤 요소 이후 생성(both의 경우 두 가지 동시에 대응하기 위해 both 사용) content:" "; display:block; clear:both; }

header를 float로 띄워서 header:: after { content:" "; display:block; clear:both; } 로 쓰는 게 국룰

Flex

예전에는 float로 다 띄우고 요소:last-child에 margin:0을 줘서 가로 배치 했는데 이게 너무 비효율적이라 나온 것이 Flexbox

flex container : display:flex;를 선언한 부모 요소
item : 컨테이너의 자식 요소(배치 시킬 요소)
main axis : 가로(row) 배치의 주축/세로(column) 배치의 보조축
cross axis 세로(column)의 주축/가로(rox) 배치의 보조축

부모 요소에 flex를 적용하면 자식 요소들이 전부 float 효과를 얻음
여기에 {justify-content:space-between; } 속성을 주면 컨테이너의 너비 전체에 item를 배치하고 item의 사이 사이에 같은 비율로 공백을 줌 (한글의 글 맞춤 옵션과 비슷한 효과인듯?)
.container { display:flex; justify-content:space-between; }
